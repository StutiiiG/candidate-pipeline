Architecture

How the multi-agent pipeline is designed

Agent 1 — Intake agent

Receives raw JD + candidate text. Normalizes into a structured JSON schema: requirements, strengths, gaps, match signals, red flags. Acts as the context assembly layer — downstream agents never touch unstructured text.

Agent 2 — Scoring agent

Receives structured output from Agent 1. Scores fit across 4 dimensions (technical, systems thinking, ownership, domain). Returns a calibrated verdict and confidence level. Prompt is carefully engineered for consistent, bias-reduced scoring.

Agent 3 — Synthesis agent

Receives profile + scores. Produces a hiring brief: summary, top strengths, key risks, 4 targeted interview questions, and recommended next step. Questions are generated against the candidate's specific gap profile.

Memory layer

Every evaluation is persisted with full structured output. Enables cross-candidate ranking, feedback loops, and — in a production system — score calibration based on downstream conversion data. Currently uses localStorage; swap for Postgres + pgvector for semantic candidate search.

Stack

Node.js + Express backend (API key stays server-side). Server-Sent Events for real-time pipeline streaming. Vanilla JS frontend, no build step. Deploy to Railway, Render, or any Node host in one command.
